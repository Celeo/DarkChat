DarkChat Documentation

Overview

DarkChat was created to solve the shortcomings of HeroChat, which DarkTide used for over a year. As I remember, HeroChat was not correctly letting the Admins set permissions to channels, nor letting us correctly set channel mods. In addition, I do not believe that it, at the time, it allowed for any sensible way for players to purchase their own in-game channels, as the plugin's command system was disfunctional and manually editing its configuration was not sensible.

Features

DarkChat is a fully-featured chat overhaul, including full permissions configuration to multiple channels, full resistriction including mutes, kicks, and bans, varied channels with colors, a sensible and simple command system, logging on command and per channel/player, and private messaging. As with all of the Dark* plugins, it was developed to be used in upholding rules and easily as possible.

Configuration

When the plugin is ran for the first time, it will copy its default configuration out from the plugin .jar file into the appropriate directory. This configuration comes pre-packaged with 9 channels, defaulting to interfacing with DarkClans for faction integration. All configuration editing can be done with the in-game commands, but DarkChat does support manual editing and reloading with the appropriate command.

Commands and Permissions

All basic commands are started with any one of the following labels:
1. ch
2. dc
3. darkchat
Then, the parameters are as such, with their matching permissions:

Command Action Permission(s)
who Returns the other players in the focused channel None
list Lists the joinable channels None
join [channel] Attempts to add the player to that channel. Fails if they have insufficient permissions, the channel doesn't exist, or they are already in it. None if public channel, otherwise, that channel's permission
leavel [channel] Removes the player from that channel, so that they no longer
receive messages to it 		None
-info [channel] Shows information about a channel None
ignore [name] Toggles ignoring that player None, but ignoring does not work on players with darkchat.nopmsbypass
-nopms Stops incoming private messages from people without darkchat.mod darkchat.mod
-ghost Shows the user all chats, including channels they're not in and
private messsages. Toggle. 		darkchat.mod
-mute [channel] [player] Mutes target player in channel darkchat.mod OR being a mod of the target channel
-shadowmute [target] Player sees their own messages, but no one else does. Server-wide ignore
towards that player. Effective against raging, as they believe that they are still getting their rage out, but no one else sees it. 		darkchat.admin
-kick [channel] [player] Kicks target player from channel darkchat.mod OR being a mod of the target channel
-ban [channel] [player] Kicks and then bans (prevents re-joining) of target player in channel darkchat.mod OR being a mod of the target channel
-reload Reloads all in-game data from the configuration file. Used for manually edits. darkchat.admin
-flush Overwrites all data in the configuration file with the in-game settings. darkchat.admin
-create [channel] Loads the channel from the configuration file darkchat.admin
-logs [player] (day) Creates a text file on the server from the chat database darkchat.admin
-readout Returns information about the in-game data darkchat.admin
-defaultconfig Overwrites the configuration file with the default from the .jar darkchat.admin
-databasedamnit Forces the setup of the database darkchat.admin
-logging Starts logging all chat to a text file on the server, in addition
to storing it in the database 		darkchat.admin

Such that an example command would be /dc join general.
Sending a message to a channel without setting it as your focus is done with /dc [channel [message...].


Private message commands are as follow:
1. /tell [player] (message)
2. /whisper [player] (message)
3. /message [player] (message)
4. /msg [player] (message)
5. /t [player] (message)
A reply command, /r [message] also exists, to reply to the last person who messaged you.
If the (mesage) parameter is omitted, the command user's default chat channel with switch to a private message channel between the user and the target.
